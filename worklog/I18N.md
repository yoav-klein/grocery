# Internationalization
---

I want the application to be in both Hebrew and English.
A user can switch languages, and all the menus and texts of the application will be changed.

Additionally, I want a tenant to have a "default language", which will affect the default catalog.

So what I want is:
* A change language button in the end of the header (before the user avatar) (V)
* An interceptor that changes the locale (V)
* Messages for all the menu items and all text

## Status
Started working on all the messages. This is a no-brainer work, so I currently leave it for later.


## Cross-cutting concerns
* We need to investigate and find out what's the best way to build URLs for JavaScript. This issue emerged when I realized that sending a request to `?lang=en` or `?lang=he` caused that the string returned by `document.baseURI` includes these request parameter, which then broke the JavaScript requests which used it to construct the request URI.
